Angular 19

Component-based structure:

Components are the main building blocks of Angular applications. Each component represents a part of a larger web page.

Every Angular component has a few main parts:

  • @Component decorator – Contains metadata and configuration used by Angular.
  • HTML template – Defines the view and controls what renders into the DOM.
  • CSS selector – Specifies how the component is referenced in HTML (e.g., <app-example>).
  • TypeScript class – Contains logic and behavior such as handling user input or making API requests.

Component's HTML and SCSS can be difined in seperate files using 'templateUrl' and 'styleUrl'

To use components, we have to import them in the component's TypeScript file

Signal

Reactivity - Use for creating and managing states

Example:

This is the signal {{ count() }}

Angular tracks where signals are read and when they're updated. The framework uses this information to do additional work, such as updating the DOM with new state. This ability to respond to changing signal values over time is known as reactivity.

Dynamic Templates

Data binding

  • Interpolation: {{ sample }} – Displays component data in the template.
  • Property Binding: [src]="imageUrl" – Binds DOM properties to component values.
  • Event Binding: (click)="handleClick()" – Responds to user events.
  • Two-way Binding: [(ngModel)]="username" – Keeps input and model in sync.

Single Page Application (SPA)

SPA is a web application that load a single HTML page (index.html) and dynamically change the content without reload the whole page while navigating

Uses client-side routing (via Angular Router) to display different components/views without making a full page refresh.

How SPA works:

When the app starts, Angular loads all the required assets (HTML, CSS, JavaScript) once.

Navigation between "pages" is handled by Angular's Router Module, which shows/hides components based on the URL.

Eager loading and Lazy loading

Eager loading:Angular load all modules immediately when the app start

Lazy loading:Angular load modules only when needed
